# 章节完整性检查清单

## 使用时机
完成阶段1-2的信息收集后，在输出前对每个章节执行此检查。

## 检查项

### 1. 必答问题覆盖
对照 `key_questions.md` 中的必答问题表，逐项确认：
- [ ] 该章节涉及的所有必答问题是否已回答？
- [ ] 回答是否具体明确（非模糊表述）？

### 2. 关键数字完整性
- [ ] 涉及天数、金额、比例的地方是否有具体数字？
- [ ] 计算公式是否完整？

### 3. 法律依据标注
- [ ] 是否标注了法律名称和条款？
- [ ] 来源URL是否已记录？

### 4. 实务要点
- [ ] 是否包含操作流程？
- [ ] 是否标注常见误区或风险？

## 缺失处理

**如发现缺失 → 触发阶段3补充搜索**

记录缺失项：
```
【待补充】
章节：[章节号]
缺失问题：[具体问题]
搜索关键词：[针对性关键词]
```

补充完成后重新检查。
