# 研究流程总则

## 1. 按模块逐项推进
- 按照手册结构和模块范围开展研究，不得无序发散
- 围绕既定模块逐个完善，确保覆盖员工管理全流程

## 2. 先官方后扩展
**对每个模块，必须优先完成官方信息研究**：
- 政府官网发布的官方说明、公告、通知、办事指南
- 官方法律法规原文
- 官方监管机构发布的实施细则、问答口径、执法说明
- 官方统计数据

⚠️ 在官方资料未研究充分前，不得直接依赖媒体文章、二手解读或非官方总结形成结论。

## 3. 法律原文重点研读
获取官方法律法规后，提炼：适用范围、企业义务、员工权利、触发条件、操作时点、合规风险、处罚或法律后果

---

## 4. 三阶段研究流程

### 阶段1：官方法规收集
**目标：** 收集所有模块的官方法律法规

**搜索策略：**
- 关键词：`[国家] + labor law + official + [年份]`
- 关键词：`[国家] + [模块] + regulation + ministry`
- 示例：`UAE Federal Decree-Law 33 of 2021`

**输出要求：**
- 法律名称和编号
- 关键条款原文
- 生效日期
- **立即存储到对应章节**

---

### 阶段2：权威解读收集
**目标：** 收集权威机构的实务解读和操作指南

**搜索策略：**
- 关键词：`[国家] + [模块] + guide + [权威机构]`
- 示例：`UAE overtime calculation KPMG guide 2026`
- 优先来源：四大、律所、政府指南

**输出要求：**
- 计算公式
- 操作流程
- 实务案例
- **立即存储到对应章节**

---

### 阶段3：细节补充（按需触发）
**触发条件：** 完成阶段1-2后，检查每个章节：
- [ ] 必答问题是否全部回答？
- [ ] 关键数字是否明确？
- [ ] 操作流程是否完整？

**如有缺失 → 进入阶段3**

**搜索策略：**
- 针对具体缺失问题搜索
- 示例：`UAE probation period notice requirement`
- 交叉验证多个来源

**停止条件：**
- ✓ 细节已补充完整
- ✗ 搜索后仍无答案 → 标注"⚠️ 需本地律师确认"

---

## 5. 信息存储规范

⚠️ **强制要求：每完成一个模块的搜索，立即用 Edit 工具将结果追加到草稿文件，不得积压。**

草稿文件路径：`~/.claude/skills/global-employment-compliance/[国家]_draft.md`

每个模块写入格式：

```markdown
## ch[XX] [章节名] ✅/🔄

**法律依据：** [法律名称和条款]
**关键内容：** [提炼的要点]
**数字/公式：** [具体数字]
**来源：** [URL]
**缺口：** [仍需补充的内容，无则写"无"]
```

状态标记：
- `🔄` = 搜索中/有缺口
- `✅` = 本章研究完成，可进入输出

**为什么必须落地：** auto compact 会无预警压缩对话内容，草稿文件是唯一不受 compact 影响的信息载体。如果 compact 发生，可直接读取草稿文件恢复进度，无需重新搜索。
