# 增量更新机制

## 适用场景

当用户请求"更新XX国手册的XX模块"而非生成全新手册时，使用增量更新模式。

## 增量更新流程

### 1. 读取现有手册
```
读取最终文件 → 提取目录结构 → 定位目标章节
```

### 2. 识别关联章节

法律变更通常影响多个章节。使用以下交叉引用表定位所有需要同步更新的位置：

| 变更类型 | 主章节 | 关联章节（必须同步检查） |
|:---------|:-------|:----------------------|
| 最低工资调整 | 4 薪酬 | 3 招聘（Offer薪资）、7 合同（合同条款）、14 风险（违规处罚） |
| 社保费率变更 | 6 社保 | 4 薪酬（雇主成本）、5 个税（扣除项） |
| 签证政策变更 | 3 招聘 | 6 社保（外籍员工规则）、13 外汇 |
| 解雇法规变更 | 8 合同解除 | 7 合同（试用期）、10 工会（纪律处分）、14 风险 |
| 假期制度变更 | 9 假期 | 4 薪酬（年假折现）、8 合同解除（未休年假结算） |
| 数据保护法变更 | 11 数据合规 | 3 招聘（背调）、14 风险 |
| 税率/税法变更 | 5 个税 | 4 薪酬（税前税后）、6 社保（扣除项） |
| 公休日变更 | 9 假期 | 4 薪酬（加班费计算） |
| 退职金规则变更 | 8 合同解除 | 4 薪酬（普通工资范围）、7 合同 |

### 3. 搜索最新法规

针对变更内容执行精准搜索：
```
[国家] + [具体法律名称/条款号] + [年份] + amendment/update
```

### 4. 输出 diff

不重写全文，仅输出变更部分：

```markdown
## 更新记录 [YYYY-MM-DD]

### 变更1：[章节号] [变更标题]

**变更前：**
> [原文内容]

**变更后：**
> [新内容]

**变更依据：** [法律名称、条款号、生效日期]

### 变更2：[关联章节号] [同步更新标题]

**变更前：**
> [原文内容]

**变更后：**
> [新内容]

**变更依据：** 与变更1关联
```

### 5. 应用变更

用 Edit 工具逐项应用到手册文件，每项变更后验证上下文一致性。

### 6. 更新版本号

在手册头部更新版本号和日期：
- 小修（数字更新、笔误修正）：V2.0 → V2.0.1
- 中修（单模块法律更新）：V2.0 → V2.1
- 大修（多模块重大变更）：V2.0 → V3.0

## 交叉引用标记规范

在手册中使用 HTML 注释标记交叉引用，便于增量更新时定位：

```markdown
<!-- XREF: 影响章节 4.1, 7.1, 14.1 -->
## 4.1 最低工资

2026年最低工资：时薪₩10,320...
```

生成新手册时，在以下位置添加 XREF 标记：
- 最低工资章节
- 社保费率章节
- 退职金计算章节
- 签证薪资门槛章节
- 年假折现章节
- 任何包含具体数字且可能年度更新的章节
